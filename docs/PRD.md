# PRD — 스윙 자동매매 시스템

## 1. 목적 및 범위

### 목적
KOSPI/KOSDAQ 국내 주식 시장에서 스윙매매(보유기간 2일~3주)를 자동화하는 개인용 트레이딩 시스템 구축.

### 범위 (In Scope)
- 키움 OpenAPI+ 기반 실시간 시세 수신 및 자동 주문 실행
- 기술적 지표 기반 매수/매도 신호 생성
- ATR 기반 손절 및 트레일링스탑 자동 관리
- 켈리/하프켈리 포지션 사이징
- vectorbt 파라미터 최적화 및 백테스트
- 텔레그램 실시간 알림
- SQLite 매매 일지 기록

### 범위 (Out of Scope)
- 해외 주식, ETF 레버리지 매매
- 알고리즘 이외 수동 개입 주문 인터페이스
- 웹 대시보드 (Streamlit은 선택적 추가)
- 클라우드 배포

### 3.3 종목 스크리닝 (FR-SCREEN)

| ID | 요구사항 | 기본값 |
|----|---------|--------|
| FR-SCREEN-01 | 일 거래대금 필터 | ≥ 50억원 |
| FR-SCREEN-02 | 시가총액 필터 | ≥ 300억원 |
| FR-SCREEN-03 | 주가 필터 | 1,000원 ~ 500,000원 |
| FR-SCREEN-04 | 관리종목/거래정지 제외 | 자동 제외 |

| FR-SCREEN-06 | 스크리닝 결과 상위 N종목 선택 | 기본 30종목 |

### 3.4 매매 신호 (FR-SIGNAL)

**매수 신호 조건 (AND 조건)**
- 20일 이평선 위에 종가 위치 (추세 확인)
- MACD 히스토그램 양전환 (모멘텀 진입)
- RSI 40~65 구간 (과매수 아닌 구간)
- 거래량 20일 평균 대비 1.5배 이상 (거래량 확인)
- 60분봉 단기 추세 상향 (멀티타임프레임 확인)

**매도 신호 조건 (OR 조건)**
- ATR 기반 손절가 이탈
- 트레일링스탑 발동 (고점 대비 ATR × 2 하락)
- MACD 데드크로스 (수익 구간에서)
- 목표가 도달 (진입가 × 1.08 기본)
- 보유 최대 기간 초과 (기본 15 영업일)

### 3.5 주문 실행 (FR-ORDER)

| ID | 요구사항 |
|----|---------|
| FR-ORDER-01 | 시장가/지정가 주문 지원 |
| FR-ORDER-02 | 분할 매수: 1/2 진입 → 추가 확인 후 나머지 1/2 |
| FR-ORDER-03 | 체결 확인 후 미체결 시 재주문 로직 (3회 한도) |
| FR-ORDER-04 | 동시 보유 최대 종목 수 제한 (기본 5종목) |
| FR-ORDER-05 | 동일 종목 재진입 쿨다운 (기본 3 영업일) |
| FR-ORDER-06 | 장 시간 외 주문 시도 차단 |

### 3.6 리스크 관리 (FR-RISK)

| ID | 요구사항 | 기본값 |
|----|---------|--------|
| FR-RISK-01 | 종목당 최대 투자 비율 | 자본의 15% |
| FR-RISK-02 | 포지션 사이징: 하프켈리 기반 | 켈리값 × 0.5 |
| FR-RISK-03 | 종목당 최대 손실 허용 | -5% |
| FR-RISK-04 | 일일 최대 손실 한도 | -3% |
| FR-RISK-05 | 일일 한도 초과 시 당일 매매 중단 | 자동 halt() |
| FR-RISK-06 | 누적 MDD -20% 도달 시 전략 중단 및 알림 | 수동 재개 필요 |
| FR-RISK-07 | 주문 전 사전 리스크 체크 (pre_check) | 모든 주문 필수 |

### 3.7 백테스트 (FR-BT)

| ID | 요구사항 |
|----|---------|
| FR-BT-01 | vectorbt 기반 신호 백테스트 |

| FR-BT-03 | 파라미터 그리드 서치 (MACD/RSI 파라미터) |
| FR-BT-04 | 성과 지표 출력: 수익률, MDD, 샤프지수, 승률, 손익비 |
| FR-BT-05 | 결과 HTML 리포트 저장 |
| FR-BT-06 | Walk-Forward 검증 지원 |

|----|---------|--------|

| FR-NOTIFY-02 | 매수 체결 | 주문 체결 이벤트 |
| FR-NOTIFY-03 | 매도 체결 (수익/손실 포함) | 주문 체결 이벤트 |

| FR-NOTIFY-06 | 매매 중단 (halt) | -3% 도달 시 |

## 4. 비기능 요구사항

| 구분 | 요구사항 |
|------|---------|
| 성능 | 신호 생성 → 주문 전송 지연 ≤ 1초 |
| 안정성 | 연결 끊김 시 자동 재연결 (최대 5회 재시도) |
| 로깅 | 모든 주문/체결/오류 영구 기록 (SQLite + 파일) |
| 보안 | API 키는 .env 파일만 사용, 코드에 하드코딩 금지 |
| 테스트 | Paper Trading 모드 필수 검증 후 실거래 전환 |
