```bash

```

- [ ] 현재 버전 상태 확인
- [ ] 필요 시 백업 실행
- [ ] 배포 알림 발송

### 2단계: 배포 실행

```bash
# 배포 명령어
{{deploy_command}}

# 예시
# vercel --prod
# kubectl apply -f deployment.yaml
# docker-compose up -d
```

- [ ] 배포 명령 실행
- [ ] 배포 진행 상황 모니터링
- [ ] 배포 로그 확인

### 3단계: 검증

```bash

# 버전 확인

```

- [ ] 헬스체크 엔드포인트 응답 확인
- [ ] 배포된 버전 확인
- [ ] 핵심 기능 수동 테스트

## 롤백 절차

### 롤백 트리거 조건

- [ ] 헬스체크 실패 지속 (3회 이상)
- [ ] 에러율 급증 (기준: > 5%)
- [ ] 핵심 기능 장애
- [ ] 성능 급격한 저하 (응답 시간 2배 이상)

### 롤백 실행

```bash
# 롤백 명령어
{{rollback_command}}

# 예시
# vercel rollback
# kubectl rollout undo deployment/{{app_name}}
# docker-compose down && docker-compose -f docker-compose.prev.yml up -d
```

### 롤백 후 조치

- [ ] 롤백 완료 확인
- [ ] 서비스 정상화 확인
- [ ] 장애 원인 분석
- [ ] 포스트모템 작성
